# GEMINI.md - ERP RBAC-RAG System Documentation

## Directory Overview

This directory contains the complete design, architecture, and implementation documentation for a secure and scalable ERP (Enterprise Resource Planning) system that uses a Retrieval Augmented Generation (RAG) model with Role-Based Access Control (RBAC). The system is designed to provide secure access to documents and information within an ERP context.

## Key Files

- **`ARCHITECTURE.md` and `ERP-RBAC-RAG-ARCHITECTURE-FINAL.md`**: These files describe the high-level architecture of the system, including the main components, their interactions, and the overall data flow.
- **`IMPLEMENTATION_GUIDE.md`**: This file provides a step-by-step guide for implementing the system, including the technology stack, project structure, and phase-by-phase implementation plan.
- **`CERBOS-INTEGRATION-GUIDE.md`**: This file details the integration of Cerbos, a policy-as-code authorization solution, into the system for managing access control.
- **`ERP-RAG-USE-CASES.md`**: This file outlines various use cases and scenarios for the RBAC-RAG system in an ERP environment.
- **`ERP-SECURITY-PATTERNS.md`**: This file describes security best practices and patterns for the system.
- **`ERP-DEPLOYMENT-GUIDE.md`**: This file provides instructions for deploying the system.
- **Other `ERP-*.md` files**: These files provide detailed information on specific aspects of the system, such as authorization policies, database connectors, document classification, performance optimization, and compliance.

## Usage

This directory serves as a comprehensive knowledge base for developers, architects, and administrators involved in the project. It provides all the necessary information to understand, implement, and maintain the ERP RBAC-RAG system.
